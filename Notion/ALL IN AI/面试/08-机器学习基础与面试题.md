# Transformer 架构详解

## 一、Transformer 架构

### 1.1 编码器与解码器

Transformer 是深度学习历史上最重要的架构之一，它彻底抛弃了 RNN，**只靠注意力机制打天下**。

---

#### 🏗️ 整体架构概览

```
输入序列 → [位置编码] → 6× 编码器层 → 6× 解码器层 → 输出
```

- 原版 Transformer 由 **6 个编码器 + 6 个解码器** 堆叠而成
- 6 层结构相同，但各层权重 **不共享**（每层学到不同抽象级别）

---

#### 🔧 编码器（Encoder）

每个编码器层 = **Self-Attention 层 → 前馈神经网络（FFN）层**

```
输入 → [自注意力] → [残差 + 层归一化] → [FFN] → [残差 + 层归一化] → 输出
```

- **自注意力层**：帮助当前词汇关注句子中所有其他词汇，获取上下文语义
- **残差连接**：缓解深层网络梯度消失问题
- **层归一化**：加速训练收敛
- **前馈网络**：两层线性变换 + ReLU，对每个位置独立计算同样的变换

> 输入输出形状不变：`(batch_size, seq_len, embedding_dim)` → `(batch_size, seq_len, embedding_dim)`

---

#### 🔧 解码器（Decoder）

每个解码器层 = **带掩码 Self-Attention → Cross-Attention → FFN**

```
输入 → [掩码自注意力] → [残差+归一化] → [交叉注意力] → [残差+归一化] → [FFN] → 输出
```

解码器比编码器多了**交叉注意力层**，用于关注编码器输出的信息（类似 seq2seq 的注意力机制）。

**带掩码的注意力（Masked Self-Attention）**：
- 在 Softmax 之前，给未来位置加上**巨大负值**（如 -1e9）
- Softmax 后这些位置注意力分布为 0
- 效果：翻译到第 K 个词时，只能看第 1~K 个词，不能偷看未来的词

---

#### 📥 输入表示：Embedding + 位置编码

```
最终输入 = Token Embedding (512维) + 位置编码 (512维)
```

**Token Embedding（词嵌入）**：
- 将每个单词映射为稠密向量（原文使用 512 维）
- 本质：One-hot 向量乘以权重矩阵，相当于查表

**位置编码（Positional Encoding）**：
- 使用 **Sin 和 Cos 交替编码**，与词向量同维度
- 每个位置生成一个唯一的编码向量，直接与词向量相加

> **为什么需要位置编码？** Transformer 没有 RNN 的循环结构，Attention 本身不包含任何时序信息。如果不加位置编码，模型会把 "我打你" 和 "你打我" 当成一样的。

---

#### 🧠 自注意力机制（Self-Attention）—— 核心创新

它是 Transformer **最核心的思想**，每一步计算如下：

**第一步：生成 Q、K、V**

对每个输入向量，训练**三个不共享的权重矩阵**：

```
输入向量 × W_Q → Query（查询）向量   → "我想找什么？"
输入向量 × W_K → Key（键）向量       → "我有什么信息？"  
输入向量 × W_V → Value（值）向量     → "我实际给出什么？"
```

Q/K/V 维度通常小于输入维度（如输入 512 维，Q/K/V 取 64 维），这相当于降维压缩。

**第二步：计算注意力分数**

```
Score = Q × K^T   →  每个词对每个词的关注度分数
```

**第三步：缩放**

```
Score = Score ÷ √d_k   (d_k = 64，√d_k = 8)
```

> 为什么除以 √d_k？防止向量维度大时点积过大，Softmax 梯度集中在两端导致梯度消失。

**第四步：Softmax 归一化**

```
Attention_weights = Softmax(Score)
```

得到 0~1 的注意力分布，代表每个词对当前位置的重要程度。

**第五步：加权输出**

```
Output = Attention_weights × V
```

所有词的 Value 按照注意力权重加权求和，得到当前位置的上下文感知表示。

---

#### ⚡ 矩阵并行计算 —— Transformer 高效的关键

RNN 只能串行：T1 时刻依赖 T0，T2 依赖 T1……
而 Transformer 直接**用矩阵一次性并行计算所有位置**：

```
输入的 N×512 矩阵 × 512×64 的权重矩阵 = N×64 的 Q 矩阵
同样操作一步算出所有位置的 K 和 V
```

> **面试考点**：Transformer 为什么比 RNN 快？→ 矩阵并行 vs 串行循环，GPU 对矩阵运算有极致优化。

---

#### 👥 多头自注意力（Multi-Head Attention）

训练**多组不同的** W_Q、W_K、W_V（原文使用 **8 组**，即 8 个头）：

| 注意力头 | 侧重捕获的关系 |
|---------|--------------|
| 头 1 | 指代消解（"它"指代什么） |
| 头 2 | 时态关系（"已经" → "完成"） |
| 头 3 | 多义词消歧（"苹果"是水果还是公司） |
| 头 4~8 | 其他语法和语义关系 |

```
8 个头各自计算注意力 → 拼接全部输出 → 乘以输出权重矩阵 → 最终结果
```

多头机制让模型从**不同角度、不同子空间**理解序列，表达能力远超单头。

---

#### 🎯 Transformer 的厉害之处

| 优势 | 详细说明 |
|------|---------|
| **高度并行化** | 自注意力和全连接层都是矩阵操作，GPU 可极致加速 |
| **信息融合强** | 介于 CNN（局部视野）和 RNN（长程依赖弱）之间，每个位置直接关注所有位置 |
| **通用性极强** | 率先统治 NLP，随后扩展到 CV（ViT）、多模态，几乎无处不在 |

---

#### 🔑 可调超参数

| 参数 | 说明 | 原文取值 |
|------|------|---------|
| 词向量维度 | 表征能力与计算量的平衡 | 512 |
| 编码器/解码器层数 | 层数越多模型越深 | 6 层 |
| 注意力头数 | 多头并行捕获不同特征 | 8 |
| 隐藏层维度 | FFN 中间层大小 | 2048 |

**架构变体**：
- **无编码器（Decoder Only）** → GPT 系列
- **无解码器（Encoder Only）** → BERT 系列
- **编码器 + 解码器** → 原版 Transformer、T5

---

#### 💡 深入理解：Attention 不是一切

虽然论文标题是 "Attention Is All You Need"，后续研究发现：

- **残差连接（Residual Connection）**：去掉后深层网络无法训练
- **前馈网络（MLP/FFN）**：存储知识的关键，去掉后性能暴跌
- 注意力机制主要负责**信息交互**，FFN 负责**知识存储**，二者缺一不可

---

#### 🎯 Transformer 评价

- 对 NLP 的意义 ≈ CNN 对 CV 的意义——**开创了一个时代**
- 几乎用在了每个模态的机器学习上：文本（GPT/BERT）、图像（ViT）、视频、语音
- 局限：模型大且贵，需要海量数据和算力

> **面试口诀**：Transformer = 位置编码 + 自注意力 + 多头机制 + 并行训练，注意力满分，RNN淘汰。

---

#### 📋 面试高频考点速记

| 问题 | 核心答案 |
|------|---------|
| 为什么抛弃 RNN？ | RNN 串行训练慢，Transformer 矩阵并行，GPU 可极致加速 |
| 为什么需要位置编码？ | Attention 不含时序，不编码位置则 "我打你" = "你打我" |
| QKV 分别是什么含义？ | Query 查什么、Key 有什么、Value 给出什么 |
| 为什么除以 √d_k？ | 防止点积过大，Softmax 梯度消失 |
| Self-Attention vs Cross-Attention？ | Self: QKV 同源（编码器内部）；Cross: Q 来自解码器，KV 来自编码器 |
| 多头注意力的作用？ | 多组 QKV 各司其职，捕获不同子空间的关系 |
| 掩码注意力的作用？ | 解码时防止偷看未来词 |

## 二、预训练模型演进（BERT / GPT）

**演进路线**：

| 年份 | 模型 | 架构 | 核心创新 |
|------|------|------|---------|
| 2017 | Transformer | Encoder-Decoder | 自注意力 + 并行训练，奠基之作 |
| 2018 | GPT | Decoder Only | 自回归语言模型预训练 + 微调 |
| 2018 | BERT | Encoder Only | 双向 Transformer，掩码语言模型 |
| 2020+ | GPT-3 及后续 | Decoder Only | Scaling Law，涌现能力，大语言模型时代 |

**BERT（双向 Transformer 编码器）**：
- 使用 **Masked Language Model（MLM）**：随机掩盖 15% 的词，训练模型预测被掩盖的词
- 双向上下文：同时看左和右，理解更准确
- 典型任务：文本分类、NER、问答

**GPT（自回归 Transformer 解码器）**：
- 使用 **自回归语言模型**：从左到右逐词预测下一个词
- Decoder Only 架构，所有 token 都参与损失计算，训练效率最高
- 典型任务：文本生成、对话、代码生成

## 三、架构对比：Encoder-Decoder vs Decoder Only

| 维度 | Encoder-Decoder | Decoder Only（Causal Decoder） |
|------|----------------|-------------------------------|
| 输入注意力 | 双向（编码器） | 单向（从左到右） |
| 训练效率 | 较低 | **最高**（所有 Token 算损失） |
| Zero-shot 能力 | 需多任务微调 | **强** |
| 代表模型 | Transformer、T5 | GPT 系列、LLaMA |

**为什么大模型大部分是 Decoder Only**：
1. **Zero-shot 表现最好**：无需微调数据即可展现能力
2. **训练效率高**：Decoder Only 在所有 token 上计算损失
3. **生成任务优势**：引入双向注意力对生成无实质好处
4. **工程实现简单**：统一架构，易于大规模扩展

## 四、面试考点速记

### Transformer 核心

| 面试题 | 核心要点 |
|--------|----------|
| Transformer 为什么抛弃 RNN？ | RNN串行训练慢，矩阵操作可高度并行 |
| 什么是 QKV？ | Query（查询）、Key（键）、Value（值），自注意力三个矩阵 |
| 什么是多头注意力？ | 多组 QKV 各司其职（指代、时态、多义词），多层表征子空间 |
| 为什么 Transformer 需要位置编码？ | 无RNN，Attention本身不含时序，需 Sin/Cos 交替编码位置 |
| Self-Attention 和 Attention 的区别？ | Self: QKV 同源（编码器内部）；Attention: Q 来自解码器，KV 来自编码器 |

### BERT vs GPT

| 维度 | BERT | GPT |
|------|------|-----|
| 架构 | Encoder Only | Decoder Only |
| 注意力 | 双向 | 单向（从左到右） |
| 训练方式 | 掩码语言模型（MLM） | 自回归语言模型 |
| 擅长任务 | 理解类（分类、NER、QA） | 生成类（文本、代码、对话） |
| 代表模型 | BERT、RoBERTa | GPT-3、GPT-4、LLaMA |

### 高频追问

| 问题 | 核心答案 |
|------|---------|
| Decoder Only 比 Encoder-Decoder 好在哪里？ | Zero-shot 强、训练效率高、生成任务双向注意无实质好处 |
| 什么是涌现能力？ | 模型大到一定程度突然出现的推理能力（数学、常识等） |
| ELMo 和 BERT 有什么区别？ | ELMo: 双向LSTM拼接；BERT: Transformer双向一体 |
| 什么是 Teacher Forcing？ | 训练时用标准答案指导解码器，避免一步错步步错 |
| 什么是 Beam Search？ | 选择候选序列集综合评分，牺牲局部最优求全局最优 |
