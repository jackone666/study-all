### **一、进程与线程**
1. **进程和线程的区别？**
    - **定义**：进程是资源分配的基本单位，线程是 CPU 调度的基本单位。
    - **资源占用**：进程有独立的地址空间、内存、文件描述符等；线程共享进程的资源，仅拥有自己的栈、寄存器和程序计数器。
    - **开销**：进程创建 / 销毁、切换的开销大；线程的开销小。
    - **通信方式**：进程间通信（IPC）需要借助管道、消息队列、共享内存等；线程间可直接读写进程数据段。
2. **进程的几种状态及转换？**
    - 基本状态：**就绪态**（等待 CPU 调度）、**运行态**（占用 CPU 执行）、**阻塞态**（等待资源或事件，如 IO）。
    - 状态转换：
        - 就绪 → 运行：CPU 调度器选中该进程。
        - 运行 → 就绪：时间片用完，或被高优先级进程抢占。
        - 运行 → 阻塞：进程发起 IO 请求或等待同步信号。
        - 阻塞 → 就绪：IO 完成或等待的信号到达。
3. **进程间通信（IPC）有哪些方式？各自优缺点？**
| **通信方式** | **优点** | **缺点** |
| --- | --- | --- |
| 管道 | 简单，匿名管道适用于父子进程 | 半双工，只能单向通信；数据在内存中，随进程结束消失 |
| 消息队列 | 独立于进程，可实现异步通信 | 有大小限制；数据拷贝开销 |
| 共享内存 | 最快的 IPC 方式，无数据拷贝 | 需要同步机制（如信号量）防止竞态 |
| 信号量 | 用于进程同步与互斥 | 只能传递信号，不能传递大量数据 |
| 套接字 | 可跨网络通信 | 开销较大，适用于不同主机进程 |
4. **什么是死锁？死锁的必要条件？如何避免死锁？**
    - **死锁定义**：多个进程因竞争资源而互相等待，且无法推进的状态。
    - **必要条件**（4 个，缺一不可）：互斥条件、请求与保持条件、不可剥夺条件、循环等待条件。
    - **解决方法**：
        - 预防：破坏任意一个必要条件（如一次性分配所有资源）。
        - 避免：银行家算法，动态判断资源分配是否安全。
        - 检测与解除：定期检测死锁，通过撤销进程或抢占资源解除。
5. **线程同步的方式有哪些？**
    - 互斥锁（Mutex）：保证同一时刻只有一个线程访问临界资源。
    - 信号量（Semaphore）：可实现多个线程的同步与互斥，支持计数。
    - 条件变量（Condition Variable）：用于线程间的信号传递，如生产者 - 消费者模型。
    - 读写锁（Reader-Writer Lock）：区分读和写，读共享、写互斥，提高并发效率。

### **二、内存管理**
6. **分页和分段的区别？**
| **维度** | **分页** | **分段** |
| --- | --- | --- |
| 目的 | 解决内存碎片，提高内存利用率 | 满足程序的模块化需求，便于共享和保护 |
| 单位 | 固定大小的页（如 4KB） | 可变大小的段（由程序逻辑决定） |
| 地址空间 | 一维地址空间（仅需页号 + 偏移量） | 二维地址空间（段号 + 段内偏移量） |
| 碎片 | 存在内部碎片（页内未用空间） | 存在外部碎片（段间未用空间） |
7. **虚拟内存的原理和作用？**
    - **原理**：基于局部性原理，利用磁盘空间模拟内存，将进程的部分数据加载到内存，其余留在磁盘，需要时再调入。
    - **作用**：
        - 扩大进程的可用内存空间，突破物理内存的限制。
        - 实现进程的内存隔离，提高安全性。
        - 允许多个进程共享内存，提高资源利用率。
8. **页面置换算法有哪些？**
    - **最佳置换算法（OPT）**：置换未来最久不使用的页面，理论最优但不可实现。
    - **先进先出算法（FIFO）**：置换最先进入内存的页面，简单但会出现 Belady 异常（增加帧数，缺页率反而升高）。
    - **最近最少使用算法（LRU）**：置换最近最少使用的页面，接近 OPT，需硬件支持记录使用时间。
    - **时钟置换算法（Clock）**：基于 LRU 的近似，用位标记页面是否被使用，开销低。
9. **什么是内存碎片？如何解决？**
    - **内部碎片**：分配的内存大于进程所需，未使用的部分在分配单元内部。解决：分页管理（尽量匹配页大小）。
    - **外部碎片**：内存中存在多个小的空闲块，总和足够但无法分配给连续需求的进程。解决：紧凑技术（移动进程合并空闲块）、分段加紧凑、分页管理。
10. **动态链接和静态链接的区别？**
    - **静态链接**：编译时将库函数代码直接复制到可执行文件中。优点：运行时无需依赖库；缺点：可执行文件体积大，库更新需重新编译。
    - **动态链接**：运行时才加载库函数，可执行文件仅保存库的引用。优点：节省内存，库更新无需重新编译；缺点：运行时依赖动态库，可能出现 “库缺失” 问题。
