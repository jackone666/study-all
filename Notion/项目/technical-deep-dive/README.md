# Enterprise Hybrid RAG — 技术深度剖析文档（分卷）

> **设计定位：规则驱动 + LLM 可选增强的 Hybrid RAG 系统。** 工作流中的路由、工具选择、意图分类等模块以关键词/规则匹配为主，LLM 调用为可选增强层（仅在非 async 上下文且配置了真实 API key 时生效）。不是真正的 LLM 自治 Agent。

本目录由原 `TECHNICAL_DEEP_DIVE.md`（6199行 / 385KB）按主题拆分而成，方便分专题阅读。

## STAR 面试答题框架

所有技术面试题已按 **STAR 法则** 结构化组织：

| 要素 | 含义 | 在文档中的标注 |
|------|------|---------------|
| **S — 情境** | 项目背景、系统环境、面临的问题场景 | `🎯 S/T/A — 项目实战（情境/任务/行动）` |
| **T — 任务** | 需要解决的具体挑战、技术要求 | 同上（与情境和行动合并） |
| **A — 行动** | 采取的具体技术方案、实现细节、设计决策 | 同上（详细代码引用 + 架构描述） |
| **R — 结果** | 方案的效果、指标、评分、经验教训 | `📊 R — 结果与评分` |

每个面试题还包含：
- **📖 通用满分答案**：独立于项目、可在其他面试中迁移的通用技术回答
- **项目评分**：基于源码实现的真实度评分（1-10分）
