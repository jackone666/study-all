## 版本 A：RAG / 大模型应用工程师岗位（推荐默认）

**Enterprise Hybrid RAG · 企业级混合检索问答系统** (Python 5K+ / LangGraph / FastAPI / 个人项目)

- 设计**三路混合检索**架构：BM25（Elasticsearch）+ 向量（Milvus）+ 知识图谱（Neo4j），按意图驱动权重路由（5 个意图 profile，70/30 融合），Weighted RRF 融合三路结果。Recall@5 从 50% 提升到 **100%**。
- 实现 **LangGraph StateGraph 工作流**（13 节点 + 7 条件路由），权限/校验/兜底全部静态边保证；`RecoveryManager` 有界重试（节点级 max_retries）+ `Tracer` 节点级埋点。P95 延迟 **6023ms → 3088ms**。
- 搭建**评估飞轮闭环**：自研 baseline 脚本输出 Recall@5/MRR/Citation/P95 等指标，发现并修复 2 个生产级 bug（LangGraph 死循环 + 异步 LLM 兜底死代码），通过率 **50% → 100%**。
- **长尾兜底引入 LLM Agent**：`ToolSelectionNode`（function-calling 选工具填参数，3s 硬超时）+ `RepairDecisionNode`（verify 失败前最后救援，最多 1 步），**主路径仍是 Workflow 保 SLA**，工具召回率 88% → 100% F1 90%。
- 中文检索 Jaccard tokenizer 加 CJK 字符级 + bigram，无外部分词依赖，性能不退化。

**技术栈**：Python 3.12, LangGraph, LangChain, FastAPI, Milvus, Elasticsearch, Neo4j, PostgreSQL, Redis, OpenTelemetry, Docker Compose, React + TypeScript

## 版本 B：LLM Agent / Agentic AI 工程师岗位

**Enterprise Hybrid RAG · Workflow + Agent 混合企业知识库系统** (Python 5K+ / LangGraph / 个人项目)

- 设计 **Workflow + Agent 双轨架构**：主路径 LangGraph StateGraph 保 SLA（P95 <3s、权限 100% 拦截、节点级可观测），**长尾路径**接入轻量 LLM Agent 提救回率——工业界企业级 Agent 系统的真实形态。
- 实现 **ToolSelectionNode**（LLM function-calling 兜底工具选择）：规则路径选不到工具或参数为空时降级，单次 call、3s 硬超时、JSON schema 强约束。在 mini 评估集上工具召回 88% → **100%**，F1 90%。
- 实现 **RepairDecisionNode**（verify 失败前最后救援）：单次 LLM call、5s 超时、硬限 1 步、不能跳权限。决策动作受限于 `retry_retrieve / retry_with_alt_query / escalate` 三种。
- 实现**三路混合检索**：BM25 + 向量 + 图谱，意图驱动权重路由（5 profile × 70/30 融合），Recall@5 从 50% → **100%**。
- 搭建**评估飞轮**：自研 before/after 对比脚本，量化每个 Agent 增量贡献，发现 + 修复 2 个生产级 bug（LangGraph 死循环 + 异步 LLM 死代码）。

**技术栈**：同版本 A
