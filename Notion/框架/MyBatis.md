### **7. #{}和 ${}的 区 别 ?**
- **编译方式**：`#{}` 是**占位符**，进行**预编译**处理；`${}` 是**拼接符**，直接进行字符串替换，没有预编译。
- **处理细节**：MyBatis 处理 `#{}` 时，会将参数作为字符串传入，把 SQL 中的 `#{}` 替换为 `?` 号，并调用 `PreparedStatement` 的 `set` 方法赋值。
- **安全性**：`#{}` 能有效**防止 SQL 注入**，提高安全性；`${}` 不能防止 SQL 注入。
- **替换位置**：`#{}` 的变量替换发生在 **DBMS（数据库管理系统）中**；`${}` 的变量替换发生在 **DBMS 外**。

### **10. MyBatis 是否支持延迟加载？原理？**
- **支持范围**：支持 `association`（一对一）和 `collection`（一对多）关联对象的延迟加载。
- **配置方式**：在 MyBatis 配置文件中，通过设置 `lazyLoadingEnabled=true|false` 来开启或关闭。
- **核心原理**：使用 **CGLIB** 创建目标对象的**代理对象**。
    - 当调用目标方法（如 `a.getB().getName()`）时，进入拦截器方法。
    - 拦截器发现 `a.getB()` 为 `null`，则单独发送事先准备好的查询关联 B 对象的 SQL。
    - 查询完成后，调用 `a.setB(b)` 填充属性，再完成原方法的调用。
- **普适性**：不仅是 MyBatis，包括 Hibernate 在内的多数 ORM 框架，延迟加载的原理基本一致。

### **14. 说 说 Mybatis的 ⼀ 级 、 ⼆ 级 缓 存 ？**
**1. 一级缓存：**
- **实现方式**：基于 `PerpetualCache` 的 `HashMap` 本地缓存。
- **存储作用域**：`SqlSession`。
- **隔离性**：各个 `SqlSession` 之间的缓存相互隔离。
- **生命周期**：当 Session 执行 `flush`（刷新）或 `close`（关闭）之后，该 `SqlSession` 中的所有 Cache 就会清空。
- **默认状态**：MyBatis 默认开启一级缓存。

**2. 二级缓存：**
- **底层机制**：与一级缓存相同，默认采用 `PerpetualCache` 和 `HashMap` 存储。
- **作用域**：**Mapper (Namespace)** 级别。这意味着在同一个命名空间下的多个 `SqlSession` 可以共享这些缓存数据。
- **扩展性**：支持自定义存储源，例如接入 **Ehcache** 或 Redis。
- **默认状态**：**默认不打开**。
- **必要条件**：使用二级缓存的对象（实体类）**必须实现 **`**Serializable**`** 序列化接口**，以便保存和恢复对象状态。
- **配置方式**：在对应的映射文件（XML）中进行配置。
