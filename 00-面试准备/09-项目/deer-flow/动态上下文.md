format_memory_for_injection(memory_data, max_tokens=2000) 负责控制注入到 system prompt 的内容量：

==用户上下文 + 历史摘要：无条件全部注入（通常很短）==
==facts：按置信度从高到低排序，逐个填入，一旦累计 token 数超过 max_injection_tokens（默认 2000），剩余 facts 直接丢弃==
==facts → 按 confidence 降序排列 → 逐个加 token 计数 → 超预算就停，后面全扔==

#### 历史摘要压缩
先判断是否超过阈值（总 token的 80% 比如），再切分旧消息和保留消息，保护动态上下文和技能内容，压缩旧消息为 summary，最后用“清空全部消息 + summary + 保留消息”的方式重写消息历史。
压缩前：
[旧消息1, 旧消息2, 旧消息3, ..., 最近消息1, 最近消息2]
压缩后：
[summary消息, 动态上下文reminder, 最近skill内容, 最近消息1, 最近消息2]
原则
**tool 消息不默认长期保留，是因为它们通常体积大、可由摘要承载结论；只有近期上下文和对后续推理强依赖的 skill 读取结果才原样保留。**
- 动态提醒保护：`_preserve_dynamic_context_reminders()` 防止 `<system-reminder>` 被压缩到错误位置。
- 技能恢复：近期通过工具读取的技能文件内容不会因为摘要压缩而丢失。

#### 长期记忆回写
如有摘要会在摘要发生之前，紧急触发下面的流程
1. **`after_agent()` 触发**  
    Agent 一轮执行结束后进入记忆处理流程。  
    一句话：**记忆写入发生在回答完成之后，不影响当前轮回复。**
    
2. **检查 `memory.enabled`**  
    如果记忆功能没开启，直接返回 `None`。  
    一句话：**总开关关闭时，整套长期记忆机制完全跳过。**
    
3. **获取 `thread_id`**  
    优先从 `runtime.context` 取，取不到再从 LangGraph `configurable` 里取。  
    一句话：**`thread_id` 用来判断这段对话属于哪个线程。**
    
4. **读取 `state["messages"]`**  
    取出当前线程里的完整消息历史。  
    一句话：**记忆来源是 LangGraph 当前状态里的消息列表。**
    
5. **过滤消息 `filter_messages_for_memory()`**  
    只保留用户输入和最终 AI 回复，丢弃工具调用过程、工具结果等中间消息。  
    一句话：**长期记忆只关心用户说了什么，以及助手最终怎么回应。**
    
6. **检查是否同时存在用户消息和 AI 消息**  
    如果只有用户消息或只有 AI 消息，就不入队。  
    一句话：**一问一答才有足够稳定的语义用于记忆抽取。**
    
7. **检测纠错信号 `detect_correction()`**  
    判断用户最近是否说了“不对”“你理解错了”“try again”等。  
    一句话：**纠错信号告诉记忆系统：之前的信息可能需要修正。**
    
8. **检测正向强化 `detect_reinforcement()`**  
    如果没有纠错，再判断用户是否说了“对，就是这样”“完全正确”“perfect”等。  
    一句话：**正向强化信号告诉记忆系统：这种偏好或事实更值得保留。**
    
9. **获取 `user_id`**  
    调用 `get_effective_user_id()`，在当前线程上下文里捕获用户身份。  
    一句话：**提前保存 `user_id`，避免异步定时器线程里丢失用户上下文。**
    
10. **加入记忆队列 `queue.add(...)`**  
    把 `thread_id`、过滤后的消息、`agent_name`、`user_id`、纠错/强化标记一起入队。  
    一句话：**记忆不会立刻写入，而是先进入带去抖的后台队列。**
    
11. **队列去抖合并**  
    默认 30 秒内同一个 `(thread_id, user_id, agent_name)` 的多次入队会合并。  
    一句话：**短时间连续对话会合并成一次记忆更新，减少频繁 LLM 调用。**
    
12. **后台执行 `MemoryUpdater.update_memory()`**  
    定时器到期后，队列调用记忆更新器，用 LLM 提取事实并写入 `memory.json`。  
    一句话：**真正的长期记忆写入是在后台异步完成的。**
    
13. **返回 `None`**  
    `MemoryMiddleware` 不修改 `state["messages"]`。  
    一句话：**它只产生记忆更新副作用，不改变当前 Agent 上下文。**
